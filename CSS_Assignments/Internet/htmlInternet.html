Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clientsare the typical Web users Internet--connected devices (for example, your computer connected to your Wi-Fi, or your phone connected to your mobile network) and Web-accessing software available on those devices (usually a web browser like Firefox pr Chrome.)
  • Serversare computers that store webpages,sites, orapps. When a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be diplayed in the user's web browser.

So What Actually Happens?

  1. The browser goes to the DNS server and fints the real address of the server that the website lives on.
  2. The browser sends an HTTP request message to the server sends the client a "200 OK" message, which means "Permission granted! Here you go!", and the starts sending the websites files to the browser as a series of small chunks.
  3. The browser assembils the small chunks into a complete website and displays it to you!

The Big 3 - HTML, CSS, JavaScript

While the server can process information in many different languages, the files that they serve to the client are always going to be some combination of HTML, CSS, and JavaScript!
